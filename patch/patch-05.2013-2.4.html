Slyum Patch 2.4

Automatisation / Confort d'utilisation

  • Il est maintenant possible de centrer les relations d'héritage d'une des manières suivantes:
    • Avec le raccourcis : <à définir>.
    • En effectuant un clic droit sur l'héritage
    • Depuis le menu.
    • En activant le centrage automatique des nouveaux héritages depuis les paramètres.
  • Les relations sont maintenant alignées sur la grille (cette option est désactivable dans les paramètres).
  • La zone d'édition du nom d'une association est maintenant visible sur le diagramme même lorsque celle-ci n'en a aucun. Cette zone apparaît lorsque vous survolez une association.
  • Vous pouvez maintenant rendre une classe/méthode abstraite en effectuant un clique droit dessus.
  • Il est maintenant possible de changer le sens d'une association depuis son menu contextuel (clique droit).
  • Le projet le plus récent s'ouvre maintenant avec Slyum.
  • Les projet s'ouvrent maintenant beaucoup plus rapidement.
  • Des repères visuels ont été ajoutés dans les sliders des propriétés.

Représentation des diagrammes de classes

  • La zone d'attribut / méthode d'une classe est maintenant complètement masquée lorsque vous décidez de ne pas l'afficher.
  • Vous pouvez mainteant choisir le mode d'affiche (quel compartiment est visible) de toutes les classes en effectuant un clique droit sur le diagramme de classe. Une option semblable est disponible dans le menu.

Corrections de bug

  • Correction du bug qui faisait s'ouvrir le menu contextuel sur OS X lorsqu'on maintenait la touche Ctrl et qu l'on effectuait un click gauche (sur une relation?).
  • Les cardinalités se placent maintenant plus proche de l'extrémité des relations.
  • Slyum ne se ferme plus sans demander de confirmation lorsqu'il y a une modification non enregistrée sur OS X.
  • La phase d'ajout de multi-association fonctionne maintenant correctement.
